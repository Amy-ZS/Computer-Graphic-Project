### Peer Testing Results

We got 5 students by using the groupme and the Canvas messaging system

We did not explain anything to the subjects besides its a fluid based physics simulation and sent them a google forms that has the following questions:

### How intuitive were the controls?
On average, our subjects said it was 8/10. This was a little polarizing though but I think part of the problem is our font that we used. It somewhat blends into the environment which makes it harder for people to read the controls.

### What was the most confusing or frustrating part of the game?
They mostly talked about the performance of the game when the render is happening.
I'm pretty sure it is still running more than once so the performance is still pretty bad.
That is what we need to fix for the final bug fix

### Did you ever not know where to go or what to do next?
Some mentioned it was a little confusing. We have a lot of controls so that you can customize your simulation.
I think we probably need a help button to help explain to the player what is happening

### What's one thing we should fix?
All our testers mentioned that we need to do more optimizations which I agree with. They also mentioned that the the name of the exe should be changed (this can definitely be fixed, we origianlly named it 'fluid.exe')


I think on lower end systems, our project is not able to get to 60fps at 10000 particles but it definitely can handle 3000. My laptop (with a 4060) can handle 10000.
